## Source chosen for phase 1

The first production slice was narrowed to:
- platform: **Pinduoduo (PDD)**
- report type: **shop-daily sales**
- initial shop: **`pdd_5`**

Sample file used:
- `/Volumes/DataHub/ecommerce/raw/pdd/shop-daily/pdd_5_daily_2026.xlsx`

Important clarification:
- `pdd_5` means **shop 5 under PDD**
- the workbook itself does **not** contain a shop identifier
- shop identity must therefore come from file metadata / config

## Schema decision

The earlier lightweight schema was replaced with a more correct v1 warehouse shape:
- `platforms`
- `shops`
- `import_files`
- `fact_shop_day_sales`

The old schema approach was rejected because it mixed grains and did not match the real PDD report shape.

## Database setup

- PostgreSQL database created/used: `edp`
- Schema successfully applied
- Verified tables exist in `edp`

## ETL implementation

OpenCode generated the first ETL script:
- `etl/load_pdd_shop_daily.py`

The script:
- parses the workbook
- converts Excel serial dates
- maps Chinese headers to canonical fields
- upserts platform/shop metadata
- records the import file
- loads shop-daily facts into `fact_shop_day_sales`

One bug was encountered during real load:
- `psql` failed on the multi-line `\copy` command
- the script was patched to use a single-line `\copy`

After the fix, the ETL load succeeded.
